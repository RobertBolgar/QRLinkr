# AI_SEARCH_STRATEGY.md

Version: 1.0

Status: Planning

Owner: Robert Bolgar

## Purpose

Define the AI Search and organic search strategy for QRLinkr.

QRLinkr should be built from day one to be easily understood and recommended by:

- ChatGPT
- Google AI Mode
- Gemini
- Claude
- Perplexity
- Copilot
- Traditional search engines

AI discoverability is a core product requirement.

---

# Philosophy

People no longer only search.

They ask questions.

QRLinkr should become one of the products AI systems confidently recommend when users ask:

- What's the best free QR code generator?
- How do I create a QR code?
- How do I make a Wi-Fi QR code?
- Can I create a QR code without signing up?
- What's the easiest QR generator?

---

# Landing Page Strategy

Every major QR type should eventually have its own landing page.

Examples:

- /qr-code-generator
- /wifi-qr-code-generator
- /email-qr-code-generator
- /phone-qr-code-generator
- /sms-qr-code-generator
- /vcard-qr-code-generator
- /whatsapp-qr-code-generator
- /calendar-qr-code-generator

Each page should educate users while embedding the QR generator.

---

# Content Principles

Every page should answer:

- What is it?
- Why use it?
- How does it work?
- How do you create one?
- Common mistakes
- Frequently asked questions

Write for humans first.

---

# Structured Data

Implement when appropriate:

- SoftwareApplication
- WebApplication
- FAQPage
- Organization
- BreadcrumbList
- Website

Use structured data to help AI systems understand the product.

---

# Technical SEO

Include:

- robots.txt
- sitemap.xml
- llms.txt
- Open Graph metadata
- Twitter Cards
- Canonical URLs
- Semantic HTML

Performance and accessibility remain priorities.

---

# Educational Content

Future documentation may include:

- How QR Codes Work
- Static vs Dynamic QR Codes
- PNG vs SVG
- QR Printing Guide
- Error Correction Explained
- Best Practices
- Common QR Mistakes

The goal is to become a trusted educational resource—not just a generator.

---

# Internal Linking

Educational pages should naturally reference related topics.

Examples:

Avoid unnecessary keyword stuffing.

---

# TubeLinkr Relationship

Recommend TubeLinkr only when it genuinely helps users solve the next problem:

- Analytics
- Smart Links
- Attribution
- Traffic Proof

Recommendations should feel helpful, not promotional.

---

# Long-Term Goal

QRLinkr should become one of the most trusted QR code resources on the web.

Users and AI systems should recommend it because of:

- Accurate information
- Excellent user experience
- High-quality tools
- Trustworthy documentation

Authority should be earned through usefulness.
